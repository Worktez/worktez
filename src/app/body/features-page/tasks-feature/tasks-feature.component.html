Create New Task

The Create text with a plus icon in the box in the header displays a pop-up when clicked. The pop-up has to be provided with details:

  • Title
  • Team
  • Task Description
  • Assignee
  • Creator (Automatically takes the current user’s email address)
  • Task Reporter: The observer of the task
  • Task Type : The types are categorised as Bug, Story, Sub-task, or Learning.
  • Sprint Number
  • Priority: Based on the urgency of the tasks it can be High, Medium and Low
  • Story Point
  • Status: Helps to determine whether the required task has been initiated or completed. It can be updated under the following terms:
    1. Ice Box: Tasks, bugs or stories having a low priority.
    2. Ready To Start: Tasks that have been assigned and now been initialised.
    3. Under Progress: Tasks or stories that are under development.
    4. Blocked: The idea behind this term is to freeze a task until the problem with it is resolved.
    5. Completed: The issues finally resolved are placed under this heading.
  • Estimated Time: The approximate time to complete the assigned task.
  • Difficulty: Low, Medium or High with respect to the task’s level of difficulty.

The above details are easily customisable in the Team Details page according to the requirements.

Task Details

The task details page displays all the details like status, priority, task id, difficulty, reporter, estimated, logged and remaining times, work progress, task description, and the links section to display the links added to task. The upload document allows the user to upload a document of any format to the task for reference.

The github icon on the left displays a pop-up and allows the user to select a PR from the integrated github repository to link it with the task.

The note icon with plus symbol displays a pop-up to log the time the user worked on a task, status of the task, and a comment about what they worked on. The user can change the status of the task according to the work done.

The three dots icon shows a dropdown on clicking which displays the options - Edit task, Clone Task, Add Link, Add SubTask, and Delete Task.

The edit task allows the user to edit the task details, clone task allows the user to clone the task or duplicate it. The add link allows the user to add any type of link to the task for reference, the add sub task allows the user to create a sub task from existing task and delete task allows the user to delete the task.

The right side of the task details page contains assign and creator names and watchers. The user can add the watchers by clicking on the eye icon. It also contains a section which displays logwork, edit, comment activity and a comment box to provide any comments.

Task Evaluation

The purpose of the task evaluation is to identify the tasks that must be completed, the deadline to be allotted to each assignment, and the person in charge of completing each task. The sprint dropdown in the task evaluation section contains all the tasks that may be dragged and dropped into another sprint, moving the work to the new sprint.

Notifications

The bell icon on the left side of the profile picture on the header displays all the notifications to the user. The notification will be displayed to the user when a task is created, a task is edited, a comment is given in the task, or work is logged in the task. The user can also navigate to their respective tasks through the notifications.
